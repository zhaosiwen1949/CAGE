# align_floorplan.py — 房间墙线对齐 / 消除房间间空隙

## 1. 背景与目标

`infer_pointcloud.py` 用 RoomFormer **逐个房间**独立预测多边形，因此相邻房间之间
**留有缝隙**，且不少墙与水平 / 竖直方向有几度偏差（见 `*_pred_floorplan.png`）。

`align_floorplan.py` 是一个**后处理脚本**：用点云本身的墙体信息，把各房间墙线
**对齐到真实墙位置**、把**近轴墙拉成严格水平 / 竖直**，从而让相邻房间**共享同一条墙**，
缝隙自然消失。对齐之后还会**细分被模型合并的房间**：用「贴顶带」结构墙 mask 检测
房间内部的隔墙，按严格判据把一个多边形切成多个真实房间（见 2.4，`--no_split` 关闭）。
它**不依赖 torch / 模型**，只复用「点云 → 密度图」的共享管线（`util/pointcloud.py`）。

**输入**：`{name}_polys.json`（`infer_pointcloud.py` 的输出）+ 对应 `.ply` 点云。
**输出**（写到 `--output_dir`，默认与 JSON 同目录）：

| 文件 | 内容 |
| --- | --- |
| `{name}_aligned_polys.json` | 对齐后的房间多边形（`pixel` + 重算的 `world_mm/world_m`），沿用原 schema，另加 `align_info` |
| `{name}_aligned_floorplan.png` | 对齐后户型图（1000×1000，彩色分房间，默认标注 room 序号） |
| `{name}_mask.png` | 墙 mask（亮=墙，256×256） |
| `{name}_split_mask.png` | **贴顶带结构墙 mask**（房间细分用的证据图，见 2.4；`--no_split` 时不生成） |
| `{name}_density_hist.png` | **调试图**：非零密度值降序曲线 + 选中阈值 + 墙像素数 |
| `{name}_aligned_overlay.png` | 对齐后墙线（绿）叠在墙 mask（灰）上，默认标注 room 序号，供人工核对吸附质量 |

> **room 序号标注**：两张图默认在每个房间上写出它在 `_aligned_polys.json` `rooms`
> 数组里的下标（0 起），两图序号一致，便于对照排查。标签锚点用 `cv2.distanceTransform`
> 取房间**离墙最远的内点**（非顶点质心，避免 L 形/凹房间标签落到墙外）。
> 加 `--no_room_labels` 可关闭。

## 2. 算法原理

### 2.1 重建「与多边形逐像素对齐」的密度图

对齐必须在与多边形**同一张栅格**上进行，否则 mask 与房间对不上。
关键点：**不重新估计任何东西**，而是复用 JSON `normalization` 里已存的量：

1. `preprocess_xyz(ply, up_axis, pct_*, crop_iqr_k)`：读点云 → `reorder_up_axis`
   （把竖直轴移到第 3 列）→ 高度百分位带 + 楼面径向 Tukey 栅栏裁飞点。
2. `rotate_floor_plane(xyz, applied_yaw_deg)`：用 **JSON 里存的 yaw**（不是重新估计）转正。
3. `density_fixed_norm(xyz, min_coords, max_coords, image_res, hflip)`：复刻 `generate_density`
   的 ps 变换与分箱公式，但用 **JSON 里存的（已含 0.1 padding 的）min/max** 分箱；
   `hflip=True` 时对成品图做 `np.fliplr`（左右翻转，见 2.1.1）。

因为 yaw 与 min/max 都取自生成该 JSON 时的值，重建的密度图与 JSON 多边形**逐像素对齐**。

> 前提：`--pct_low/--pct_high/--crop_iqr_k` 需与生成 JSON 时一致（裁剪决定哪些点入图）。
> yaw 与缩放歧义已由「复用存储值」消除。

#### 2.1.1 左右镜像修复（`hflip`，`--no_floor_hflip` 关闭）

y-up 点云的 `reorder_up_axis` 用 `[0,2,1]`（**奇置换**，翻手性），密度图相对实际户型图**左右镜像**
（"从下往上"看）。修复是对密度图做 `np.fliplr` 转成常规俯视：

- **本脚本默认输出俯视帧**（y-up 时 `hflip=True`；z/x-up 不翻）。
- JSON `normalization.hflip` 标记该 JSON 的多边形在哪个帧：`infer_pointcloud.py` 新版写入；
  **旧 JSON 无此字段视为未翻**。当"输入帧 ≠ 输出帧"时，脚本先把多边形列 `255-col` 转到目标帧，
  再重建（同样翻转的）密度图 → mask 与多边形仍逐像素对齐。
- 反投影 `pixel_to_world(hflip=True)` 用 `255-col` 反算，使 `world_mm` 与不翻时**逐位相同**，
  故 `polys_to_3d.py` 零改动仍能把结果叠回原始 `.ply`。

### 2.2 密度图 → 墙 mask

密度图里**亮 = 墙**（竖直墙面的点投影密集）。阈值都在**非零密度像素**上求
（若把全图 0 背景一起算，会退化成「0 vs 非 0」）。阈值以上即墙：

```
mask = density_u8 >= threshold      # 亮=墙
```

三种求阈值方法（`--mask_method`）：

- **`otsu`（默认）**：最大化类间方差。直方图双峰清晰时墙线最细最干净；
  代价是弱墙可能断线。
- **`knee`**：排序降序密度曲线的**膝点**——把曲线两端连一条弦，取曲线离弦最远的点
  （Kneedle 思想，免求导、无参数）。阈值比 Otsu 宽松（本场景 61 vs 98），
  墙更粗但**更连续**，家具散点噪声略多（对吸附无碍：吸附求整列/整行和的 argmax，
  相干墙线总能压过散点）。直方图偏态/单峰、或 overlay 里看到墙线断裂时用它。
  注意不要直接对曲线求一阶/二阶导数找膝点：曲线是 256 级量化的阶梯函数，
  且峰值归一化在开头造出陡崖，实测 `max|d²|` 会命中悬崖且对平滑带宽极敏感。
- **`percentile`**：`--mask_percentile 80` 固定百分位，手动兜底。

`_density_hist.png` 把非零密度降序画出来并标注选中阈值，供核对落点。

### 2.3 对齐（核心，纯在 256 像素空间进行）

记每个房间多边形顶点为像素 `[col, row]`。分四步：

**a0. 小斜边收角（`--collapse_diag_len`，默认 20）** —— 模型偶尔把直角预测成一两条小斜边
（切角毛刺，如窄小房间的角）。对每个多边形找**连续斜边段**（相邻若干条 D 边），
若段的**路径总长 ≤ 阈值**，则用两侧轴边隐含的直角替换：
- 两侧轴边相互垂直（V…H / H…V）→ 收成**一个直角角点**（两轴线交点）；
- 两侧轴边平行（V…V / H…H）→ 收成一段**垂直于它们的短桥**（桥的位置取段内顶点均值，
  之后照常被 mask 吸附修正）。

总长超过阈值的斜边段（真实斜墙，如八边形客厅的切角）不动。收角在分类/聚类**之前**做，
新生成的边会作为普通轴边参与后续拉直与吸附。

> **为什么默认开启(20)**:它不只是消毛刺,还负责把「**接近轴向但略超 `angle_tol`**」的短边拉直。
> 例如某房间阶梯墙那条与竖直差 9.5°(>`angle_tol=8`)的短边会被判为 D、**不参与 x 吸附**而停在原位,
> 与相邻房间吸到的墙错开 → **房间间 overlap**。收角把这类短边转成干净轴边后一并吸附,消除错位。
> 关掉(`--collapse_diag_len 0`)会让 room 5 类缺口和 room 0/1 类 overlap 重新出现。

> **对齐后再收一次角**：短斜边常常是**吸附把两侧墙拉正之后**才"暴露"成角上明显的斜边
> （吸附前两侧墙还略斜、被判成 D，短斜边所在的 D 链更长、两侧非轴边，a0 抓不到）。
> 因此在下面 d 步吸附完成后**再跑一次同样的收角**，把这类新暴露的短斜边收成直角。

**a. 边分类** —— 对每条边算它与坐标轴的锐角：
- ≤ `angle_tol` → **水平墙 H**（约束其两端点**共享 y**）
- ≥ 90°−`angle_tol` → **竖直墙 V**（约束两端点**共享 x**）
- 其余 → **斜边 D**（保留，不参与轴对齐，例如八边形房间的 45° 角）

**b. 并查集聚坐标** —— 全场景所有顶点建两套并查集：
- 每条 V 边 → 在 **X** 并查集里 `union` 其两端点；把这两点标记为 *x-约束*。
- 每条 H 边 → 在 **Y** 并查集里 `union`；标记为 *y-约束*。

同一组的顶点最终会取**同一个 x（或 y）**——这一步同时完成了「把近轴墙拉直」。

**c. 跨房间合并 + mask 引导定标** —— 把各 X 组聚成簇，每簇吸附到真实墙中线：

1. **二维聚类（必须同时满足两条）**：两组并入同一簇,当且仅当
   ①代表坐标相差 ≤ `snap_tol`(在吸附轴上相近)**且**②两组在**垂直方向的覆盖区间重叠/相接**
   (缝隙 ≤ `snap_tol`)。后者是关键:坐标相近但垂直区间不相交的两道墙,是楼里**不同位置**的
   墙(如左半楼与右半楼在相近 y 的两道墙),不能并簇。只按坐标做 1D 单链会把整层同一 y
   的墙全串成一簇,把吸附拽向墙多的那半楼。
2. **局部 mask 定标（连续性感知）**：每簇在其吸附轴值域 `[lo−pad, hi+pad]` 内选目标行/列。
   两个要点：
   - **局部统计**：墙像素只在该簇覆盖的垂直区间内数（`mask[c, x0:x1]` 而非整行 `mask[c, :]`）——
     否则该行别处的无关墙会主导 `argmax`、把吸附点拉偏。
   - **去噪计数（拒绝断线）**：候选线的得分不是原始墙像素数，而是**只统计属于长度 ≥
     `wall_min_run` 的连续段的像素**。真墙是连续直线;而家具/杂物投影成的**断断续续**的列
     (若干 2~3px 碎段)即便总数不低，也**明显不是墙**——去噪计数把这类碎段归零、直接淘汰。
     注意是用连续性**排除假墙**，而非「最大化连续段」:后者会让吸附跳到窗口内**另一条**恰好
     更长的平行墙(相邻房间的墙)。真墙之间再按总像素数、就近簇均值决出。
   - mask 在该范围无信号时退化为簇均值。Y 组同理取行、按 x 区间统计。

> 因为相邻两个房间**共享的那道墙**在 mask 里是同一条亮带，两边的墙坐标(垂直区间重叠)
> 会聚进同一簇、吸附到同一列/行 → **共享墙坐标，缝隙消失**；而不相邻、只是恰好坐标相近的
> 墙各自独立吸附到自己的真实墙(例如阶梯状的相邻墙线会各归各位，不被强行拉平成一条)。

**d. 重写顶点 + 无损化简** —— 每个顶点：*x-约束* 则取其 X 簇目标、否则保留原 x；y 同理。
于是斜边的顶点只在「它确实接了轴边」的那个轴上被吸附，纯斜角两坐标都原样保留。
最后做**三层化简清理**（迭代到不动点，紧跟对齐后收角之后）：

1. 连续重复点；
2. **近共线 / 回折点**（`--collinear_tol`，默认 2px）：删除任何到「两邻点连线 A–C」的
   **垂距 ≤ collinear_tol** 的顶点 B（`|cross(B−A,C−B)| / |C−A|`）。既覆盖精确共线的
   多余中点、精确回折刺（零面积毛刺），也覆盖**浅回折缺口的尖点**——比如斜墙向外凸出
   一两像素又折回（垂距仅 ~1.5px），这类点既不是精确共线（规则旧版 `==0` 抓不到）、
   又被 a0 当作长斜边链的一部分放过。删除它只把轮廓移动 ≤ collinear_tol px。
   `A、C` 重合（纯回到原点的刺）按垂距 0 处理。`--collinear_tol 0` 退回只删精确共线。
3. **锐角尖刺**（`--spike_angle_deg`/`--spike_max_gap`）：顶点内角 < `spike_angle_deg`
   **且**两邻点开口 ≤ `spike_max_gap` px → 视为针状尖刺（外凸/内凹皆可）删除尖点。
   处理「离邻点连线很远（垂距大）、但基座极窄」的细长刺——这类刺规则 2 抓不到，靠它兜底。

退化（<3 点）则回退为原多边形。真实斜墙/切角因垂距远大于 `collinear_tol`、且不是锐角，
不受这三条影响。

#### 为什么锐角规则必须带「开口守卫」

**开口守卫（`--spike_max_gap`）不能省，不能只凭锐角删点。** 根本原因：**锐角只反映
「尖不尖」，不反映「大不大」**，而噪声尖刺与真实尖角房间在角度上无法区分，只能靠**尺寸**
（开口 `|AC|`）区分：

- 噪声尖刺：尖 **且** 基座只有几像素（如 4px 宽、30px 长的针）——这个尺度不可能是真实墙，删掉安全。
- 真实尖角：细长三角房、尖头飘窗、非正交地块的斜墙角，可以很尖 **但** 基座是几十像素的真实结构。

若去掉守卫、只要锐角就删，会出现：

1. **真实尖角房间被削顶**：三角房尖点（内角十几度、开口二三十像素）被删 → 三角房退化成线段、尖头凸窗被切平。
2. **斜墙角被误并**：斜墙与轴墙（或两条斜边）相交处若恰好是锐角，被删 → 相邻两段墙合并、房间变形。
   （xinghewan 八边形的角都是 127°~158° 钝角才幸免，是运气不是安全。）
3. **量级是结构级而非噪声级**：删一个开口 25px 的尖点会把轮廓一次挪动几十像素；而规则 2 删点只挪 ≤ `collinear_tol`（≤2px），是无损级。
4. **级联**：删掉一个锐角顶点后相邻角度改变，可能连锁触发、越删越多，啃进真实墙线。

两条化简规则各自都**自带尺寸约束**，这才是安全的关键：规则 2 按**垂距**约束（挪动 ≤ 容差），
规则 3 按**开口**约束（只删噪声尺度的基座）。开口守卫让规则 3 **故障安全**——最坏是留一个
看得见的小噪声刺（调大 `--collinear_tol` 或 `--spike_max_gap` 即可清），而不是**静默删掉一个
真实墙角**。「户型图无锐角」只是经验假设（斜楼、尖头飘窗、非正交地块都可能有真实锐角），
即便接受它，真出现的锐角多半是大结构、误删就是大变形，故守卫仍不可省。

> 有较宽的噪声缺口没清掉时，优先调大 `--collinear_tol`（按垂距、无损）或适度调大
> `--spike_max_gap`，**不要**取消守卫。

### 2.4 房间细分（split，`--no_split` 关闭）

模型常把几个小房间预测成**一个**多边形（如 衣帽间+卫生间、过道+衣帽间、门厅并入大厅）。
对齐完成后，本步骤检测每个房间**内部的隔墙**并把房间切开，递归进行。

#### 双 mask：为什么用「贴顶带」找隔墙

主 mask（全高度带）里门洞是断的、吧台/橱柜/家具会成噪声，不适合直接找内墙。
`ceiling_band_mask` 只投影**天花板附近**（默认高度范围的 `[0.75, 0.95]` 分位段）的点：

- **墙到顶**：隔墙在贴顶带里是连续亮线；
- **家具不到顶**：吧台、橱柜台面、矮柜在这一带消失；
- **门洞被过梁封口**（若 MVS 重建出了门上方；实测星河湾点云门上常缺失，故判据允许一个门洞缺口）；
- 上界必须 < 1.0：**天花板本身是水平面**，含进去会把整张图淹成全亮。
- 阈值用非零密度的 `--split_mask_percentile`（默认 80）分位——贴顶带直方图不双峰，Otsu 不可靠。

**分位段扫带实测**（星河湾，以 4 刀真墙全对、零误切为判准）：安全区
`lo ∈ [0.65, 0.82]`、`hi ∈ [0.92, 0.98]`，区内真墙覆盖率几乎不变；默认
`[0.75, 0.95]` 取余量居中。两侧失败模式：`lo` 太低 → 中段家具/杂物线混入产生误切；
`lo ≥ 0.85` 或 `hi = 1.00` → 墙线变薄漏切，且最顶部薄片含**逐房间高度不一的
吊顶/天花板残面**，成片泛白污染判据。**过梁弱成像与分位段基本无关**：门洞上方
raw 密度多数非零（点是有的），但比墙体稀疏 1 个量级、达不到阈值；只有含
`hi=1.00` 的顶片才点亮门洞，代价是天花板泛白，不可取——所以门洞感知缺口判据
（而非调带）才是正解。

#### 下刀判据（宁漏分、不误分）

对每个房间栅格化后，沿 x / y 扫描每条候选切割线（两端各留 `--split_min_size`），
**全部满足**才允许切：

1. **内部弦**：只统计「线两侧沿切割轴各有 ≥ `split_min_size` 深度房间面积」的像素
   （1D 腐蚀实现）。房间自己的**台阶边界墙**（如飘窗前沿正好骑在底边界墙厚度里）
   不得冒充隔墙证据。
2. **双端接界**：首/末墙像素距弦两端 ≤ `--split_end_gap`（默认 3px）。
   独立式烟道/高柜/吧台线最多一端搭墙，被此条拒绝（实测厨房内 55% 跨度单端线）。
3. **覆盖率**：连续性感知计数（只算 ≥ `--wall_min_run` 的连续段）≥ `--split_min_cover`
   （默认 0.5）。
4. **门洞感知缺口**：线上每个空洞只能是 **噪声**（≤2px，最多 1 个）或
   **门洞尺寸**（`[--split_door_min, --split_door_max]`，默认 [7, 24]px，最多 1 个）。
   3~6px 的洞（≈0.3–0.6m）现实墙上不存在——那是淋浴隔断/杂物线（实测卫生间A 内
   6px 洞的假线被此条拒绝）；碎成多段小洞的（窗框/窗帘盒）也被拒绝。
5. **全高交叉验证**：同一条线在**主 mask** 里的去噪覆盖率也须 ≥ `--split_main_cover`
   （默认 0.4，0=关闭）。真隔墙**从地到顶**、两张 mask 都亮（实测三条真隔墙两 mask
   覆盖率逐位相同）；**窗帘盒/吊顶边只存在于贴顶带**（主 mask 0.22）、
   **到顶衣柜前脸**在低处被杂物打碎（主 mask 去噪覆盖 0.00），都被此条拒绝。

通过判据的最强线用 shapely 半平面相交切开（两个子房间**共享切割线坐标**，
与共享墙约定一致，零缝隙零重叠），子多边形化简后**递归**再检。任一子块 bbox
短边 < `--split_min_size`（默认 8px）则放弃该刀。切割记录写入
`align_info.splits`（parent/axis/line/cover），子房间带 `split_from` 字段
（= 切割前的 room 下标；注意切割会**改变房间编号**）。

**切割后再跑一遍对齐（2.3）**：切割线是各房间在贴顶 mask 上独立取的 argmax，
不经过跨房间坐标聚类；真墙带有 3~4px 厚时，切割线与邻房已吸附的墙线可能都"在墙上"
却相差几个像素（实测门厅切割线 y=68 vs 相邻过道D 底边 y=65），形成台阶。第二遍
`align_rooms` 把这些轴对齐的新边与邻房坐标聚成一簇、统一吸到主 mask 最密行/列，
台阶（及其退化出的薄舌头）随化简消失。

> 星河湾实测：13 → 17 房间，4 刀全对（衣帽间B|卫C、过道B|衣帽间A、卫D|过道D、
> 门厅|餐厅客厅），厨房/卫生间/卧室零误切；飘窗前沿、到顶衣柜两类假线分别被
> 判据 1 和 5 拒绝。

### 2.5 反投影与落盘

对齐后的像素多边形用 `pixel_to_world(pixel, min_coords, max_coords)`
（`world = min + pixel/255 · (max−min)`）重算 `world_mm`，`world_m = world_mm/1000`，
写回同 schema 的 JSON，并渲染上述各调试图。

## 3. 使用方法

```bash
conda activate CAGE
# 默认处理 infer_out/xinghewan_da3_mvs_polys.json + data/custom/xinghewan_da3_mvs.ply
bash tools/align_floorplan.sh

# 或指定：
bash tools/align_floorplan.sh <polys.json> <point_cloud.ply> <output_dir>

# 直接调脚本、带调参：
python align_floorplan.py \
  --polys=infer_out/xinghewan_da3_mvs_polys.json \
  --ply=data/custom/xinghewan_da3_mvs.ply \
  --output_dir=infer_out \
  --snap_tol=5 --angle_tol=8 --mask_method=otsu
```

### 参数

| 参数 | 默认 | 说明 |
| --- | --- | --- |
| `--polys` | （必填） | `*_polys.json` 路径 |
| `--ply` | （必填） | 对应点云 |
| `--output_dir` | 同 JSON 目录 | 输出目录 |
| `--pct_low` / `--pct_high` | 2 / 98 | 高度百分位裁剪，**须与生成 JSON 时一致** |
| `--crop_iqr_k` | 3.0 | 楼面径向 Tukey 栅栏，**须与生成 JSON 时一致** |
| `--mask_method` | `otsu` | `otsu`（双峰直方图最干净）/ `knee`（排序曲线弦距膝点，墙更连续）/ `percentile`（手动） |
| `--mask_percentile` | 80 | `percentile` 模式下的分位（≥ 视为墙） |
| `--angle_tol` | 8.0 | 近轴判定容差（度）。斜墙被误拉直 → 调小 |
| `--snap_tol` | 5.0 | 坐标聚类容差（像素）。缝没消干净 → 调大；不同房间被误并 → 调小 |
| `--collapse_diag_len` | 20 | 连续斜边段总长 ≤ 此像素数则收成直角（消切角毛刺 + 拉直近轴短边防错位/overlap）。0=关闭；xinghewan 毛刺段 5~17px、真斜墙 ≥ 32px |
| `--spike_angle_deg` | 60 | 内角小于此度数视为尖刺候选（户型图无锐角）。0=关闭 |
| `--spike_max_gap` | 10 | 尖刺开口（两邻点距离）上限，px。守卫真实宽楔形不被角度判据误删 |
| `--collinear_tol` | 2 | 到邻点连线垂距 ≤ 此值的顶点删除（近共线/浅回折尖点）。0=只删精确共线 |
| `--wall_min_run` | 5 | 吸附时只有属于 ≥ 此长度连续段的墙像素才算墙，拒绝断线/杂物列。调大=更严格要求连续（细分的覆盖率计数同样复用它） |
| `--no_split` | （默认开） | 关闭房间细分（见 2.4） |
| `--split_band_lo` / `--split_band_hi` | 0.75 / 0.95 | 贴顶带在（裁剪后）高度范围内的分位区间。实测安全区 lo∈[0.65,0.82]、hi∈[0.92,0.98]（见 2.4 扫带）；上界须 < 1.0 避开天花板平面 |
| `--split_mask_percentile` | 80 | 结构 mask 阈值（非零密度分位） |
| `--split_min_cover` | 0.5 | 隔墙线去噪覆盖率下限（贴顶带） |
| `--split_main_cover` | 0.4 | 全高交叉验证：同一线在主 mask 的去噪覆盖率下限。0=关闭 |
| `--split_end_gap` | 3 | 双端接界：首/末墙像素距弦端最大距离（px） |
| `--split_door_min` / `--split_door_max` | 7 / 24 | 线上合法门洞缺口的尺寸区间（px）；3~6px 的洞视为假线证据 |
| `--split_min_size` | 8 | 子房间最小 bbox 短边（px，256 栅格 ≈ 0.7~1m）；内部弦的两侧深度要求同值 |
| `--no_room_labels` | （默认标注） | 关闭 floorplan / overlay 上的 room 序号标注 |
| `--no_floor_hflip` | （默认翻转） | 关闭 y-up 的左右翻转（见 2.1.1）。z/x-up 本就不翻 |

## 4. 调参与排查

- **还有小缝没消**：调大 `--snap_tol`（如 6~8）。
- **不该合并的两道墙被并到一起**：调小 `--snap_tol`。
- **本该斜的墙被强行拉直**：调小 `--angle_tol`（如 5）。
- **相邻房间出现 overlap 或角上缺口**：多半是 `--collapse_diag_len` 被设成 0。
  近轴短边(略超 `angle_tol`)会被判为斜边、不参与吸附而错位 → overlap/缺口。
  用默认 20（或按场景把它设到 < 最短真斜墙段总长）。
- **真斜墙被收角了**：调小 `--collapse_diag_len`（该值须 < 最短真斜墙段总长）或设 0 关闭。
- **真实的窄楔形/锐角结构被删了**：调小 `--spike_max_gap` 或 `--spike_angle_deg`，
  设 `--spike_angle_deg 0` 完全关闭尖刺清理。
- **斜墙/八边形上仍有小回折缺口没消掉**：调大 `--collinear_tol`（如 3~4）。
- **真实的浅折角被拉平了**：调小 `--collinear_tol`（如 1）或设 0（只删精确共线）。
- **墙吸到了断断续续的杂物列（明显不是墙）**：调大 `--wall_min_run`（如 6~8），更严格要求连续。
- **真墙本身较碎、被 `wall_min_run` 判成非墙而吸不到**：调小 `--wall_min_run`，或换 `--mask_method knee` 让墙更连续。
- **墙线断裂、部分墙吸不到位（退化为簇均值）**：换 `--mask_method knee`，墙更连续。
- **mask 太粗/太细或吸错位置**：`--mask_method percentile` 配合 `--mask_percentile`
  手动收紧/放宽；对照 `_density_hist.png` 与 `_aligned_overlay.png` 判断。
- **场景里有两条贴近的平行墙带（柜体/管井贴墙）**：otsu 与 knee 可能吸到不同的那条
  （细 mask 两带打平时按靠近簇均值决出，厚 mask 按更强的带决出）——用
  `_aligned_overlay.png` 人工核对，选贴合真实墙的方法。
- **mask 与房间明显错位**：多半是 `--pct_*/--crop_iqr_k` 与生成 JSON 时不一致。
- **该分的房间没分开（漏分）**：对照 `_split_mask.png` 看隔墙线是否成像。线在但没切 →
  降 `--split_min_cover`（如 0.4）或放宽 `--split_door_max`；线本身断碎 → 调宽贴顶带
  （如 `--split_band_lo 0.70`）或降 `--split_mask_percentile`。门洞特别宽的垭口
  （> door_max）会被拒，属有意保守。
- **`_split_mask.png` 大片泛白**：`--split_band_hi` 太高（吃进吊顶/天花板残面），压回
  ≤ 0.97；**墙线太细/断**：`--split_band_lo` 太高，降到 0.70~0.75。
- **单一房间被误切（误分）**：升 `--split_min_cover` / `--split_main_cover`，或
  `--split_min_size` 调大（切出的假"房间"往往是 ≤1m 的薄条：衣柜、飘窗、管井）。
  确认误切来源可对照 `_split_mask.png`（贴顶带假线）与 `_mask.png`（是否全高）。
- **切割后房间编号变了**：细分按原房间顺序展开子房间，编号必然移位；用子房间的
  `split_from` 字段回溯切割前的下标。
